PluginManager Klasse

Die PluginManager-Klasse ermöglicht das Laden, Entladen und Verwalten von Plugins.

Eigenschaften

  • Instance: Statische Eigenschaft, die eine Singleton-Instanz des PluginManagers zurückgibt.

Methoden

  • LoadPlugins(string pluginDirectory): Lädt alle Plugins aus dem angegebenen Verzeichnis.
  • LoadPlugins(IEnumerable<string> dllPaths): Lädt Plugins aus einer Liste von DLL-Pfaden.
  • LoadPluginByName(string pluginName, string pluginDirectory): Lädt ein bestimmtes Plugin aus dem angegebenen Verzeichnis anhand des Namens.
  • UnloadPluginByName(string pluginName): Entlädt ein bestimmtes Plugin anhand des Namens.
  • UnloadAllPlugins(): Entlädt alle geladenen Plugins.
  • Dispose(): Gibt Ressourcen frei und beendet den PluginManager.

Verwendung des PluginManager

  1. Importieren Sie das PluginManager-Namespace.
  2. Verwenden Sie die Instance-Eigenschaft, um auf die Singleton-Instanz des PluginManagers zuzugreifen.
  3. Verwenden Sie die verschiedenen Methoden des PluginManagers, um Plugins zu laden, zu entladen oder Informationen darüber zu protokollieren.

Erstellung eines Plugins

  1. Erstellen Sie ein neues Projekt für das Plugin und implementieren Sie das IPlugin-Interface aus dem WPInterfaces-Namespace.
  2. Kompilieren Sie das Plugin-Projekt in eine DLL-Datei.
  3. Platzieren Sie die DLL-Datei im Plugin-Verzeichnis, das Sie beim Laden des Plugins angeben.

Das war die API-Dokumentation und Anleitung zur Verwendung des PluginManagers sowie zur Erstellung von Plugins.
